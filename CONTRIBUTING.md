# Contributing

Thank you for helping make SP404 Toolkit reliable and transparent.

1. Use Node.js 20+ and install dependencies with `npm install`.
2. Run `npm test`, `npm run typecheck`, and `npm run build` before proposing a change.
3. Add tests for every parser, writer, mapping, or validation change.
4. Never commit copyrighted commercial samples.
5. Use synthetic fixtures and document how each fixture was produced.
6. Mark reverse-engineering statements as `CONFIRMED`, `OBSERVED`,
   `HYPOTHESIS`, or `UNKNOWN`.
7. Do not describe a binary field as confirmed without repeatable evidence.

Architectural decisions with lasting impact belong in `docs/decisions`.
